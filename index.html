Sistema de Cadastro de Usuários - Automation Exercise

Informacoes do Card

ID: US-001 Prioridade: Alta Story Points: 8 Sprint: 1

Historia de Usuario

Como um novo visitante do site Automation Exercise,
Quero poder criar uma conta de usuario com meus dados pessoais,
Para que eu possa acessar funcionalidades exclusivas, fazer compras e receber ofertas personalizadas.

Descricao Detalhada

Esta funcionalidade permite que novos usuarios se cadastrem no sistema atraves de um fluxo completo de registro. O usuario fornece informacoes basicas (nome e email), preenche dados da conta (senha, data de nascimento) e informacoes adicionais (endereco, telefone). O sistema deve validar as informacoes, criar a conta e confirmar o sucesso do cadastro.

Objetivo de Negocio

Aumentar a base de usuarios registrados e coletar informacoes relevantes para marketing e personalizacao da experiencia de compra.

Requisitos Tecnicos

  • Validacao de email unico no sistema
  • Criptografia de senha
  • Campos obrigatorios e opcionais claramente identificados
  • Suporte a multiplos paises no cadastro de endereco
  • Sistema de opt-in para newsletter e ofertas

Criterios de Aceite

Criterio 1: Acesso a Pagina de Cadastro

Dado que estou na pagina inicial do site,
Quando clico no botao "Signup / Login",
Entao devo ser redirecionado para a pagina de cadastro/login e visualizar o formulario "New User Signup!".

Criterio 2: Validacao Inicial (Nome e Email)

Dado que estou no formulario "New User Signup!",
Quando preencho um nome valido e um email valido que nao existe no sistema,
Entao devo avancar para o formulario completo "ENTER ACCOUNT INFORMATION".

Criterio 3: Preenchimento de Dados da Conta

Dado que estou no formulario "ENTER ACCOUNT INFORMATION",
Quando preencho todos os campos obrigatorios (titulo, senha, data de nascimento),
Entao devo poder selecionar opcoes de newsletter e ofertas especiais.

Criterio 4: Preenchimento de Dados de Endereco

Dado que preenchi os dados da conta,
Quando preencho informacoes de endereco completo (nome, sobrenome, empresa, endereco, pais, estado, cidade, CEP, telefone),
Entao devo poder finalizar o cadastro clicando em "Create Account".

Criterio 5: Confirmacao de Cadastro

Dado que submeti o formulario completo de cadastro,
Quando o sistema processa as informacoes com sucesso,
Entao devo visualizar a mensagem "ACCOUNT CREATED!" e poder continuar.

Criterio 6: Login Automatico

Dado que minha conta foi criada com sucesso,
Quando clico em "Continue",
Entao devo estar automaticamente logado e visualizar "Logged in as [nome do usuario]".

Criterio 7: Exclusao de Conta (Opcional para Teste)

Dado que estou logado no sistema,
Quando clico em "Delete Account",
Entao minha conta deve ser removida e devo visualizar "ACCOUNT DELETED!".

Resultado Esperado

Resultado Geral

Sucesso: O fluxo completo de cadastro de usuario deve ser executado sem erros, desde o acesso a pagina inicial ate a confirmacao de exclusao da conta.

Resultados Especificos por Etapa

Etapa 1-3: Acesso ao Site
  • A pagina inicial do Automation Exercise carrega completamente
  • Todos os elementos visuais (logo, menu, banners) estao visiveis
  • Nao ha erros de carregamento ou mensagens de erro
Etapa 4-5: Navegacao para Cadastro
  • O botao "Signup / Login" e encontrado e clicavel
  • Redirecionamento para a pagina de cadastro ocorre corretamente
  • O formulario "New User Signup!" e exibido claramente
  • Campos de nome e email estao disponiveis para preenchimento
Etapa 6-8: Cadastro Inicial
  • Os campos aceitam a entrada de texto normalmente
  • O botao "Signup" fica habilitado apos o preenchimento
  • Ao clicar em "Signup", o sistema processa a solicitacao
  • Nao ha mensagem de erro indicando email duplicado
Etapa 9-16: Preenchimento de Dados da Conta
  • O titulo "ENTER ACCOUNT INFORMATION" e exibido no topo do formulario
  • Nome e email sao preenchidos automaticamente do passo anterior
  • Campos de titulo (Mr./Mrs.) estao disponiveis para selecao
  • Campo de senha aceita entrada e mascara os caracteres
  • Dropdowns de data de nascimento (dia, mes, ano) funcionam corretamente
  • Checkboxes de newsletter e ofertas sao marcaveis
  • Validacoes de campos obrigatorios funcionam (se aplicavel)
Etapa 17-26: Preenchimento de Endereco
  • Todos os campos de endereco aceitam entrada de texto
  • Dropdown de pais exibe lista de paises disponiveis
  • Campos numericos (CEP, telefone) aceitam formatacao adequada
  • Nao ha limite restritivo de caracteres nos campos de texto
  • Campo Address2 e opcional e pode ficar em branco
Etapa 27-28: Criacao da Conta
  • O botao "Create Account" esta habilitado e clicavel
  • Ao clicar, o sistema processa o cadastro (pode haver loading)
  • A pagina de confirmacao e exibida
  • A mensagem "ACCOUNT CREATED!" aparece de forma destacada
  • Botao "Continue" esta disponivel e visivel
Etapa 29-30: Login Automatico
  • Ao clicar em "Continue", o usuario e redirecionado para a pagina inicial
  • O sistema realiza login automatico do usuario recem-criado
  • No cabecalho da pagina, aparece "Logged in as [nome do usuario]"
  • O nome exibido corresponde ao nome fornecido no cadastro
  • O menu muda para refletir o estado de usuario logado
  • Opcao "Delete Account" fica disponivel no menu
Etapa 31-33: Exclusao da Conta
  • O botao/link "Delete Account" e encontrado e clicavel
  • Ao clicar, o sistema processa a exclusao da conta
  • A mensagem "ACCOUNT DELETED!" e exibida de forma clara
  • Botao "Continue" esta presente para finalizar o processo
  • Apos clicar em "Continue", o sistema retorna ao estado de usuario nao logado
  • A conta e permanentemente removida do banco de dados

Criterios de Sucesso Global

  • Nenhuma mensagem de erro e exibida durante todo o fluxo
  • Todos os redirecionamentos funcionam corretamente
  • Todas as validacoes de dados funcionam adequadamente
  • O usuario consegue completar o cadastro em menos de 5 minutos
  • A interface esta responsiva e carrega em tempo aceitavel (menos de 3 segundos por pagina)
  • Dados sao persistidos corretamente no banco de dados
  • A exclusao da conta remove todos os dados do sistema

Casos de Falha

O teste falha se:

  • Qualquer pagina nao carregar corretamente
  • Mensagens de validacao esperadas nao aparecerem
  • Elementos da interface (botoes, campos) nao estiverem visiveis ou funcionais
  • A conta nao for criada apos submissao do formulario
  • O login automatico nao funcionar
  • A exclusao da conta falhar ou nao remover os dados
  • Ocorrer qualquer erro de sistema (500, 404, timeout, etc.)
